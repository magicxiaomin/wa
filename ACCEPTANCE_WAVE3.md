# ACCEPTANCE_WAVE3 · 第三波验收

## Phase 1：发给最多 3 个联系人（wrapper）
- [ ] SendTextMulti 能发给 2 个联系人，两个都收到，各返回 server_msg_id
- [ ] 能发给 3 个联系人，全部成功
- [ ] **发给 4 个（超上限）被拒绝，返回明确错误，一条都不发**
- [ ] 多人发送之间有随机节流间隔（不是瞬间全发）
- [ ] 单个收件人失败不影响其余，结果列表逐个标明成功/失败

## Phase 2：发到群（wrapper）
- [ ] GetGroups 能返回当前账号已加入的群列表（JSON：jid/name）
- [ ] 能向选定的群发送文本，群里成员能收到
- [ ] 发群拿到 server_msg_id

## Phase 3：Android UI
- [ ] 联系人列表支持多选，且**最多只能选 3 个**（选满后其余置灰/提示）
- [ ] 选 1-3 人发送，UI 显示每个收件人的结果
- [ ] 群列表页能展示 GetGroups 结果
- [ ] 选一个群发文本，UI 显示发送结果
- [ ] UI 层和 wrapper 层的 3 人上限是双层保险（都生效）

## Phase 4：真机 + trace
- [ ] 真机上：选 2-3 个联系人发消息，他们都收到
- [ ] 真机上：选一个群发消息，群成员收到
- [ ] trace 覆盖多人发送和群发送的事件，且脱敏（不含正文/全号码/session key）
- [ ] 整个过程 risk-stop / 节流仍然生效

## 最终判定
- [ ] 在真机上：能选最多 3 个联系人发同一条消息、能选一个群发消息，都送达。
- [ ] 超过 3 人被硬拒绝（账号安全保险生效）。
- [ ] 范围守住：没有出现发给 >3 人、发到多群、多账号等越界行为。

## 不阻塞通过的已知项
- 收群消息本波可不做（建议延后）
- 国产 ROM 保活、封号观察、长时间后台运行：延续观察，不在本波解决
- 发信体验精细优化：本波不做
