## C. Non-blocking 问题

1. **`go.mod` 的 go 指令（`go 1.25.0`）与工具链 pin（`go1.26.3`）不一致。**
   - 影响：版本叙述漂移；构建固定 1.26.3，但 `go.mod` 声明 1.25.0。
   - 证据：`go.mod:3` vs `android/build_debug_go126.sh`（校验 `go1.26.3`）。
   - 最小修复：把 `go.mod` 的 `go 1.25.0` 升到 `go 1.26.3`，或在 `ANDROID_BUILD_TOOLCHAIN.md` 注明双版本含义。

2. **`WaBridgeClient` 在 `onServiceDisconnected` 后不自动重连，且 `bound` 不复位。**
   - 影响：`:wa_bridge` 进程被系统杀掉时，`service=null` 但 `bound` 仍为 true，后续调用抛 `bridge service is not connected`，集成方必须自己 `unbind()`+`bind()` 才能恢复，但 `bind()` 见 `bound==true` 会直接 return，导致无法恢复。
   - 证据：`WaBridgeClient.kt:48-50`（`onServiceDisconnected` 只置 `service=null`）、`:57-58`（`bind()` 在 `bound` 为 true 时直接 return）。
   - 最小修复：`onServiceDisconnected` 内置 `bound=false`；并在 `SDK_API.md` 注明“进程被回收后需重新 `bind()`”。可选加 `onBindingDied` 复位。

3. **`cmd/wa-poc` 桌面 PoC 仍在仓库，走 `SendTextForTest` 白名单路径。**
   - 影响：非 SDK 交付物，留在主线让“纯 Android SDK 仓库”的叙述略显混杂。使用安全白名单发送，无 cloud/远程，**不构成安全问题**。
   - 证据：`cmd/wa-poc/main.go:81`。
   - 最小修复：`README` 注明 `cmd/wa-poc` 为非交付物 PoC，或后续移除。

4. **AAR 打包仍是“手工拆包”（carryover，已注释缓解）。**
   - `wamobile-classes.jar` + `jniLibs/libgojni.so` 是从 `wamobile.aar` 拆出并提交的生成物；build 脚本已注释“改 Go 必重跑”。可接受；长期可改为直接以 AAR 依赖消费消除漂移面。

## D. API / SDK 集成风险（第三方视角）

- **进程被杀后的恢复责任全在集成方**（见 C2）：文档应给出“调用抛 `WaBridgeException` / `getState` 异常 → 重新 bind”的恢复范式。**风险中等**。
- **同步 AIDL 调用阻塞调用线程**：业务方法是同步跨进程调用，文档已提示“放后台线程”，但未明确“绝不可在主线程调用”，主线程调用会 ANR。建议在 `SDK_API.md` 顶部加醒目约束。**风险中等**。
- **`onEvent(eventType, payloadJson)` 是无类型字符串契约**：向后兼容好（加字段不破坏）；`SDK_API.md` 已逐事件给 payload 形状。**风险低**。
- **单设备/单账号是硬约束**：`@Volatile client` + `clientLock` 双检锁，符合定位；文档已写明。**风险低**。
- **ABI 仅 arm64-v8a**：`INSTALL.md` 已注明测试设备需支持 arm64-v8a；x86_64 模拟器不可用。**风险低，已披露**。

## E. 真机回归 checklist（最小集）

- [ ] 登录/恢复：`bind()` → 有 session → `getState()=connected` 触发 `session_restored`；无 session → `qr_generated` 扫码 → `connected`。
- [ ] 联系人/群：`getContacts()` 非空；`getGroups()` 列出群（jid/name/participant_count）。
- [ ] 1:1 收发：`sendText(对方,...)` 收 `message_sent`（带 `server_msg_id`）；对方回消息触发 `message_received`。
- [ ] ack 链：送达 → `message_ack` `ack_level=1`；已读 → 同 `server_msg_id` 再来 `ack_level=2`。
- [ ] 群发：`sendText(群@g.us,...)` 群成员收到，拿到 `server_msg_id`（仅验发，收群消息不支持）。
- [ ] clear→重连：`clearSession()` → `getState()` 未连接 → `connectBridge()` 能再次扫码登录，无残留。
- [ ] 进程被杀恢复：`adb shell am kill :wa_bridge` 后，集成方重新 `bind()` 能恢复，无双 client / 无 sqlite db lock（重点验 C2 恢复路径）。
- [ ] 导出路径：`exportTrace`/`exportSessionDebug` 传 `filesDir` 内成功；传 filesDir 外被拒；产物未进 git。
- [ ] 回调线程：listener 内直接 `setText` 不抛 `CalledFromWrongThreadException`。
